# Ingatlan 网站部署说明（一步一步）

把本地做好的网站放到公网，让任何人用浏览器都能访问。下面用 **Render**（国外免费托管）为例，从零开始操作。

---

## 第一步：把代码放到 GitHub

网站要部署，需要先把代码上传到 GitHub，部署平台再从 GitHub 拉代码来运行。

### 1.1 安装 Git（如果还没装）

1. 打开：https://git-scm.com/download/win  
2. 下载 Windows 版，双击安装，一路“下一步”即可。  
3. 安装好后，**关闭并重新打开** Cursor 或命令行窗口。

### 1.2 注册 GitHub 账号

1. 打开：https://github.com  
2. 点右上角 **Sign up**，用邮箱注册一个账号并登录。

### 1.3 在 GitHub 上新建一个仓库（Repository）

1. 登录后，点右上角 **+** → **New repository**。  
2. **Repository name** 随便起一个，例如：`ingatlan-platform`。  
3. 选 **Public**，下面的 “Add a README” 可以**不勾选**（我们本地已有代码）。  
4. 点 **Create repository**。

### 1.4 把本地的 ingatlan-platform 项目推送到这个仓库

在 Cursor 里打开终端（菜单 **终端 → 新建终端**），在**项目根目录**（能看到 `index.html`、`server` 文件夹的那一层）依次输入下面命令（把 `你的用户名` 和 `ingatlan-platform` 换成你在 GitHub 上的用户名和仓库名）：

```bash
git init
git add .
git commit -m "首次提交：网站与后端"
git branch -M main
git remote add origin https://github.com/你的用户名/ingatlan-platform.git
git push -u origin main
```

- 第一次 `git push` 可能会让你登录 GitHub（浏览器弹出或输入用户名/密码）。  
- 若提示要输入用户名，填你的 **GitHub 用户名**；密码处填 **Personal access token**（在 GitHub 网站：Settings → Developer settings → Personal access tokens 里生成一个）。

推送成功后，在浏览器里打开 `https://github.com/你的用户名/ingatlan-platform`，应该能看到所有项目文件。

## 第二步：在 Render 上部署

### 2.1 注册 Render

1. 打开：https://render.com  
2. 点 **Get Started for Free**。  
3. 选 **Sign up with GitHub**，用你的 GitHub 账号登录并授权。

### 2.2 新建一个 Web Service（网站服务）

1. 登录后，在 Dashboard 里点 **New +** → **Web Service**。  
2. 在 **Connect a repository** 里找到你刚推送的仓库（如 `ingatlan-platform`），点 **Connect**。  
3. 若列表里没有，点 **Configure account** 去授权 Render 访问你的 GitHub 仓库，再回来选。

### 2.3 填写配置（按下面一字不差填）

| 项 | 填什么 |
|----|--------|
| **Name** | 随便起名，例如 `ingatlan`（会出现在网址里：`ingatlan.onrender.com`） |
| **Region** | 选离你或用户近的，如 **Singapore** 或 **Oregon** |
| **Branch** | `main`（默认） |
| **Root Directory** | **留空**（不要填） |
| **Runtime** | **Node** |
| **Build Command** | `npm install` |
| **Start Command** | `npm start` |

### 2.4 环境变量（重要）

在 **Environment** 或 **Environment Variables** 里点 **Add Environment Variable**，添加：

| Key | Value |
|-----|--------|
| `NODE_ENV` | `production` |
| `SESSION_SECRET` | 随便一长串英文+数字，例如：`myIngatlan2025SecretKeyChangeThis` |

- `SESSION_SECRET` 用来加密登录状态，**不要和别人分享**，自己记牢即可。

### 2.5 开始部署

1. 点页面底部 **Create Web Service**。  
2. Render 会自动执行 `npm install` 和 `npm start`，等几分钟，下面日志里出现 “Your service is live at …” 就表示成功。

### 2.6 访问你的网站

- 页面上会显示一个地址，类似：**https://ingatlan.onrender.com**  
- 用浏览器打开这个地址，就能看到和本地一样的首页；注册、登录、搜索等都会连到这台服务器。

## 第三步：以后更新网站怎么弄？

改完本地代码后，在项目根目录执行：

```bash
git add .
git commit -m "更新说明（随便写）"
git push
```

Render 会检测到 GitHub 有更新，自动重新部署一次（在 Dashboard 里能看到 “Deploy” 进度）。部署完成后，刷新网站即可看到新内容。

## 常见问题

**Q：部署后打开网站很慢？**  
Render 免费版在没人访问一段时间后会“休眠”，第一次打开要等几十秒唤醒，属于正常现象。

**Q：没有 GitHub / 不想用 GitHub？**  
可以改用 **Railway**（https://railway.app），也是连 GitHub 部署；或把整个项目文件夹打包，用 **Railway** 的 “Deploy from folder” 上传（需先注册）。

**Q：想用自己的域名？**  
在 Render 的 Web Service 里点 **Settings** → **Custom Domains**，按提示把你的域名指过去即可。

**Q：Build 或 Start 报错？**  
在 Render 的 **Logs** 里看红色报错。常见情况：Build 没写 `npm install`、Start 没写 `npm start`，或仓库里缺少 `package.json`。确认根目录有我们提供的 `package.json`，并且 `server` 文件夹里有 `server.js` 和 `server/package.json`。
